# AI Usage Guidelines

Guidelines for TAs on handling AI-assisted code in student projects and code reviews.

## Overview

AI tools like ChatGPT, GitHub Copilot, and Claude are increasingly common in software development. Our goal is to help students use these tools responsibly while ensuring they develop genuine understanding and professional skills.

## Appropriate AI Use

### Learning and Understanding
- **Explain concepts:** "How does this algorithm work?"
- **Debug errors:** "Why is this code throwing this exception?"
- **Clarify documentation:** "What does this function parameter do?"
- **Understand best practices:** "What's the difference between these approaches?"

### Code Review and Improvement
- **Generate test cases:** Create edge cases for existing functions
- **Suggest improvements:** "How could this be more readable?"
- **Code style suggestions:** Formatting and naming recommendations
- **Documentation help:** Writing clear docstrings and comments

### Problem-Solving Support
- **Brainstorm approaches:** "What are different ways to solve this?"
- **Architecture discussions:** "How should I structure this project?"
- **Tool selection:** "What library would be best for this task?"

## Inappropriate AI Use

### Complete Solutions
- **Copy-pasting entire functions or scripts** without understanding
- **Submitting AI-generated code** as original work
- **Using AI to bypass learning objectives**

### Without Understanding
- **Can't explain the code** they've submitted
- **No comprehension** of the underlying concepts
- **Unable to modify or debug** AI-generated code
- **Missing project context** and specific requirements

### Academic Integrity Issues
- **No attribution** when AI assistance was used
- **Presenting AI work as original** without disclosure

## Detecting AI-Generated Code

### Code Characteristics
- **Overly generic:** Perfect but lacks project-specific context
- **Unnecessary complexity:** Over-engineered for simple tasks
- **Missing project specifics:** No hardcoded paths, project structure, or context
- **Inconsistent style:** Doesn't match the student's usual coding patterns
- **Perfect documentation:** Comprehensive docstrings that seem beyond the student's level

### Behavioral Indicators
- **Can't explain the logic:** Struggles to walk through the code
- **Sudden skill jumps:** Code quality dramatically improves without learning progression
- **Generic variable names:** Uses placeholder names like `data`, `result`, `temp`
- **Missing error handling:** AI often generates "happy path" code without edge cases
- **No project integration:** Code doesn't fit with existing project structure

### Review Red Flags
- **Perfect but context-free:** Code works but doesn't match project requirements
- **Overly comprehensive:** Includes features not requested or needed
- **Inconsistent with student's level:** Advanced concepts they haven't demonstrated before
- **Missing project-specific details:** No references to actual data files, paths, or requirements

## How to Address AI Usage

### During Code Reviews

#### Ask Guiding Questions
- "Can you walk me through this logic step by step?"
- "Help me understand how this function works"
- "What happens if the input data is empty?"
- "How does this fit with the rest of your project?"

#### Request Explanations
- "I'd like to understand your approach here"
- "Can you explain why you chose this method?"
- "What alternatives did you consider?"
- "How would you modify this for different data?"

#### Focus on Learning
- "Let's trace through this together"
- "What would you do if this failed?"
- "How would you test this function?"
- "What would you change if the requirements changed?"

#### Encourage Attribution
- "Did you use any AI assistance here?"
- "What resources did you consult?"
- "How did you approach solving this problem?"

### Educational Responses

#### For Appropriate Use
- **Acknowledge good practices:** "Great use of AI to understand this concept"
- **Encourage learning:** "Now that you understand this, how would you apply it?"
- **Build on understanding:** "What would you do differently next time?"

#### For Inappropriate Use
- **Focus on learning:** "Let's work through this together so you understand it"
- **Set expectations:** "For this assignment, I need to see your own work"
- **Provide guidance:** "Here's how you could approach this problem yourself"
- **Encourage growth:** "Understanding this will help you in future projects"

## Escalation Guidelines

### Immediate Escalation
- **Clear evidence** of AI-generated solutions for graded work
- **Academic integrity violations** with AI assistance
- **Repeated violations** after warnings
- **Student unable to explain** any of their submitted code

### Pattern-Based Escalation
- **Consistent use** of AI without understanding across multiple submissions
- **Multiple students** in the same team using AI inappropriately
- **Systematic issues** with AI usage policies
- **Resource constraints** preventing proper supervision

### Documentation Requirements
- **Save examples** of suspected AI-generated code
- **Note inability** to explain code during reviews
- **Record conversations** about AI usage
- **Document patterns** across multiple submissions
- **Include context** about assignment requirements

### Effective Escalation Process
1. **Document the issue** with specific examples
2. **Note learning objectives** that are being bypassed
3. **Suggest educational interventions** rather than punitive measures
4. **Include student's response** to questions about the code
5. **Recommend next steps** for ensuring learning outcomes

## Best Practices for TAs

### Prevention
- **Set clear expectations** about AI usage in project guidelines
- **Encourage appropriate use** of AI as a learning tool
- **Model good practices** by showing how to use AI effectively
- **Create assignments** that require understanding, not just code generation

### Detection
- **Ask questions** during code reviews to assess understanding
- **Look for patterns** that suggest AI assistance
- **Pay attention** to sudden changes in code quality
- **Focus on comprehension** rather than just functionality

### Response
- **Prioritize learning** over punishment
- **Use educational approaches** to address inappropriate use
- **Encourage attribution** and transparency
- **Help students understand** the value of genuine learning

## Key Teaching Points

### For Students
- **AI is a tool for learning**, not a shortcut to avoid understanding
- **Attribution is important** - be transparent about AI assistance
- **Understanding is the goal** - you should be able to explain your code
- **Professional development** requires genuine skill building

### For TAs
- **Focus on learning outcomes** rather than catching violations
- **Use AI as a teaching opportunity** to discuss professional practices
- **Encourage responsible use** while maintaining academic integrity
- **Build understanding** through guided questions and explanations


## Remember

The goal is building professional data scientists who can work effectively with AI tools while maintaining understanding, integrity, and professional standards. Help students develop the skills to use AI responsibly throughout their careers.
