# Teaching Basics

## Core Responsibilities
- Conduct two in-person TA sessions each week.  
- Track and report attendance via Canvas.  
- Support students with technical issues (GitHub workflows, debugging, Docker, etc.).  
- Ensure students adhere to professional project workflows.  
- Evaluate project repositories during finals week using the official checklist.  

## Best Practices for TA Sessions
- Treat sessions as **structured working time**, not office hours.  
- Check in with each team member during sessions.  
- Help groups prioritize tasks and divide responsibilities.  
- Encourage problem-solving strategies rather than solving issues for students.  

## Boundaries
- Do not complete student deliverables.  
- Escalate repeated unprofessional or disengaged behavior to the mentor or clinic director.  
- If unsure how to handle a situation, ask the mentor or director rather than making exceptions.  

## Checklist for Each Week
- [ ] Run two TA sessions.  
- [ ] Record attendance in Canvas.  
- [ ] Provide project-specific support and guidance.  
- [ ] Flag any concerns with student performance or professionalism.
