# Technical Skills

## Tools and Practices Students Are Expected to Learn
- **Python best practices**: modular code, documentation, testing.  
- **Git/GitHub**: branching, pull requests, code review, merging.  
- **Reproducibility tools**: Docker, Makefiles.  
- **Collaboration**: using GitHub Issues, tracking progress.  

## TA Responsibilities
- Assist with Git operations, including reviewing and merging pull requests.  
- Provide feedback on code readability, structure, and documentation.  
- Troubleshoot common issues with Python, Jupyter, Docker, and environments.  
- Help students transition from exploratory notebooks to reproducible workflows.  

## Common Technical Challenges
- Merge conflicts and broken branches.  
- Students not documenting code or using tests.  
- Difficulty containerizing code with Docker.  
- Dependency and environment management issues.  

## Examples of TA Guidance
- Demonstrating how to structure a Python package.  
- Walking through a simple Dockerfile.  
- Showing how to write clear commit messages.  
- Explaining the difference between exploratory notebooks and production code.
