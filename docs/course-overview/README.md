# Course Overview

## Course Objectives
The Data Science Clinic is designed to give students:
- Experience working on real-world, client-driven data science projects.  
- Practice in applying best practices for Python development (documentation, modular code, testing).  
- Exposure to modern data science tools (GitHub, Docker, Make).  
- Professional skills in collaboration, communication, and project management.  

## Role of TAs
TAs support these objectives by:
- Guiding students in the practical implementation of data science best practices.  
- Monitoring student engagement and accountability during TA sessions.  
- Providing technical and process support without completing the work for students.  
- Acting as a bridge between students, mentors, and the clinic director.  

## Course Structure
- Students work 10–15 hours per week on team projects.  
- Each team meets with their mentor weekly.  
- TAs hold **two weekly 1-hour TA sessions** as structured working periods.  
- Students submit weekly progress reports.  
- Projects run across two quarters, with evaluations at the end of each quarter.  

## Common Student Challenges
- Managing messy, real-world data.  
- Transitioning from one-off notebooks to reproducible workflows.  
- Learning to collaborate effectively with GitHub and pull requests.  
- Balancing client expectations with technical feasibility.  
- Staying consistent in documentation, testing, and code style.
