# Data Science Clinic – TA Training

Welcome to the Teaching Assistant (TA) training repository for the Data Science Clinic.  This handbook provides guidance, policies, and resources to help you succeed as a TA while supporting students working on real-world data science projects with clients.

## Purpose
- Equip TAs with the **pedagogical, technical, and professional skills** required in the Clinic.  
- Provide a **reference guide** you can use throughout the quarter.  
- Ensure **consistency across projects** so that every student team benefits from the same standards and practices.  

## Repository Structure

### TA Role and Expectations ([`docs/`](./docs/README.md))
These documents cover the **teaching, professionalism, and day-to-day responsibilities** of a TA.

### Technical Handbook ([`technical/`](./technical/README.md))
This section documents the **technical standards, workflows, and review practices** that TAs should enforce during projects.  

## TA Expectations

As a TA, you are expected to:  
- Conduct **two weekly TA sessions** that function as structured working periods.  
- Support students with **technical issues** (Git, Python, Docker) and reinforce best practices.  
- **Track and report attendance** weekly through Canvas.  
- **Review and merge pull requests** responsibly, ensuring repo hygiene.  
- Evaluate project repositories during finals week using the [official checklist](https://clinic.ds.uchicago.edu/rubrics/final-technical-cleanup.html).  
- Maintain professionalism and act as a role model for students.  

## Getting Started
1. Read through the [**Course Overview**](/docs/course-overview.md) and [**Teaching Basics**](/docs/teaching-basics.md) to understand your role and responsibilities.  
2. Review the [**TA Session Guide**](/docs/ta-sessions.md) for a playbook on running weekly sessions.  
3. Familiarize yourself with the [**Technical section**](/technical/README.md) — this is what you’ll enforce with students.  
4. Bookmark the [**Resources**](/docs/resources.md) section and official Clinic website for reference during the quarter.

## Key Principles of the Clinic
- **Automation**: All tasks should be automated where possible.  
- **Reproducibility**: All results must be reproducible.  
- **Documentation**: All code, processes, and data must be clearly documented.  
- **Accountability**: Students must contribute actively and professionally.  
- **Industry Alignment**: Students should learn workflows that reflect professional data science practice.  

## Contact
If issues arise that are beyond the TA scope, escalate promptly:  
- **Mentor**: First line of escalation for technical/project issues.
- **Clinic Director**: Escalation for grading disputes, unprofessional behavior, or exceptions.
